Payment API

API Endpoint

Payment API 可直接將交易資料傳送過來進行交易，交易完成後會回覆資料，中間不透過網頁傳導。優點是快速，且商店可使用自己的付款頁而不用考慮風格不同的問題。
API規格中的編碼皆為utf-8。

Payment ¶

Payment 交易傳入 ¶

商店端在結帳時將訂單交易資料以form post型式送到此API，經驗證正確後就會直接向後端發動交易，然後回覆交易結果。

支付工具為非信用卡(mmk, atm, webatm)時，因為是非同步交易(非一次動作即可得知繳費結果)，所以會在消費者繳費後另行異步通知。

通知格式請參考PayPage串接規格中的"交易結果通知"章節。

Payment 交易傳入
POST/1177payment/api/acceptpayment

Request 參數

名稱 型別 必要參數® 說明
merchantnumber String(10) R 商店編號，由1177Tech配發，只可英數字
ordernumber String(30) R 訂單編號，必須是唯一值，若有重複則會回覆訂單編號重複的錯誤，只可英數字和特殊符號[_-]。
merchantordernumber String(30) O 商店訂單編號，此欄位不做唯一值的確認，供商店做訂單的對照使用，若不帶則系統分配的預設值為訂單編號。
amount Number(11,2) R 訂單金額，可接受小數到小數點後2位，小數部分主要是為了多幣別時使用，當幣別為新台幣時接受金額只到整數，若有指定小數則無條件捨去。
currency String(3) O 幣別，預設為TWD。(目前僅接受台幣)
paymenttype String(20) O 此筆訂單所要使用的支付工具，若不帶此參數則表示到PayPage後再選擇支付工具。目前支援的參數值有creditcard, mmk, atm, webatm。
paytitle String(50) R 付款說明，此說明主要會顯示在某些支付工具的收款單位介面上，例如在超商的Kiosk機台上或是銀行WebATM的介面上。
paymemo String(200) O 付款註記，在某些應用時需要系統多註記資訊以便日後查詢使用。
duedate Number(14) O 訂單有效期限，若未帶此參數則會以各商店預設的期限為準(於商家資料設定)，有效期限會依選定的支付工具套用，例如信用卡交易可以到秒(yyyyMMddHHmmss)，超過這個時間才送出的訂單會被視為無效；而虛擬帳號因為銀行的期限到日(yyyyMMdd)，所以會自動判定為指定時間的當天，例如指定20181008153400，則此訂單在20181009即失效。
id String(10) O 身分證字號或統編。
payphone Number(12) O 消費者聯絡電話，可以是手機或市話，市話需加區碼。
timestamp Number(14) R 時間戳記，格式為yyyyMMdd，若目前時間與時間戳記相差超過3分鐘即失效。
consumerip String(15) O 消費者IP
checksum String(64) R 驗證碼，用以驗證傳送資料是否遭竄改。
cardnumber Number(16) O 卡號，當支付方式為信用卡(paymenttype=creditcard)時為必要參數。
cardexpiry Number(6) O 到期日，當支付方式為信用卡(paymenttype=creditcard)時為必要參數。格式為yyyyMM。
cardcvc2 Number(4) O 卡片背面末三碼，當支付方式為信用卡(paymenttype=creditcard)時為必要參數。visa、master、jcb時為3碼，amex(目前不支援)為4碼。

驗證碼計算方式：

將參數名稱依字母排序後，以key=value&key=value&…的格式組合好後(若沒有值則不加入驗證)，最後直接補上交易密碼(transcode，申請商店後由系統核發)，以utf-8取byte值後進行sha256。

範例： 交易密碼(transcode)為 3710F366CD0329255166A5B30216E176

資料為

amount=200.5

merchantnumber=201387

ordernumber=54613351

paytitle=這是測試

paymemo=

資料排序組合後為 “amount=200.5&merchantnumber=201387&ordernumber=54613351&paytitle=這是測試”

準備進行hash的明文為 “amount=200.5&merchantnumber=201387&ordernumber=54613351&paytitle=這是測試3710F366CD0329255166A5B30216E176”

產出的驗證碼為 “6B67D324F6A4396F0857192AE05946C443D28C7BC8E9C5CDB305F26671805A84”

名稱 型別 必要參數® 說明
rc String(4) R 回覆碼, rc=0: 成功, rc=1: 網頁需進行轉導, rc=其他: 交易失敗。
rm String(50) O 回覆訊息。
merchantnumber String(10) R 商店編號，由1177Pay配發，只可英數字。
ordernumber String(30) R 訂單編號，必須是唯一值，若有重複則會回覆訂單編號重複的錯誤，只可英數字和特殊符號[_-]。
paymentnumber Number(14) O 付款序號，以此序號確認是否為重複繳款：若收到多次通知但序號相同則為重複銷帳，不須理會；若收到多次通知且序號不同則為重複繳款，必須記錄，此與帳務相關。交易成功時才會回覆。
bankrc1 String(10) O 收單銀行Gateway回覆碼，選擇信用卡付款(paymenttype=creditcard)且交易成功時才會回覆。
bankrc2 String(10) O 發卡銀行回覆碼，選擇信用卡付款(paymenttype=creditcard)且交易成功時才會回覆。
approvecode String(6) O 授權碼，選擇信用卡付款(paymenttype=creditcard)且交易成功時才會回覆。
bankapprovetime Number(14) R 授權時間，選擇信用卡付款(paymenttype=creditcard)且交易成功時才會回覆。
ins_down_amt Number(9) O 分期首期金額，選擇信用卡分期付款(paymenttype=creditcard)且交易成功時才會回覆。
ins_each_amt Number(9) O 分期每期金額，選擇信用卡分期付款(paymenttype=creditcard)且交易成功時才會回覆。
red_offset_amt Number(9) O 紅利扣抵金額，選擇信用卡紅利付款(paymenttype=creditcard)且交易成功時才會回覆。
red_used_point Number(9) O 紅利扣抵點數，選擇信用卡紅利付款(paymenttype=creditcard)且交易成功時才會回覆。
amount Number(11,2) O 應繳金額，為捨去小數位後的實際應繳金額。
paycode String(14) O 超商繳費代碼，選擇超商付款(paymenttype=mmk)時才會回覆。
virtualaccount String(14) O 虛擬帳號，選擇虛擬帳號付款(paymenttype=atm)時才會回覆。
redirect_method String O 網頁轉導方式，選擇信用卡3D交易(paymenttype=creditcard, use3d=1)，或是網路ATM轉帳(paymenttype=webatm)時才會回覆，可能的值有post, get。
redirect_data String O 網頁轉導資料，選擇信用卡3D交易(paymenttype=creditcard, use3d=1)，或是網路ATM轉帳(paymenttype=webatm)時才會回覆。內容為JSON格式，請以key, value的型態取出後放在網頁的form中以hidden方式呈現。
redirect_url String O 網頁轉導網址，選擇信用卡3D交易(paymenttype=creditcard, use3d=1)，或是網路ATM轉帳(paymenttype=webatm)時才會回覆。
redirect_encode String O 網頁轉導編碼，選擇信用卡3D交易(paymenttype=creditcard, use3d=1)，或是網路ATM轉帳(paymenttype=webatm)時才會回覆。有些收款單位(例如銀行)可能只接受big5編碼，有些收款單位可能接受utf-8編碼。

CancelOrder 取消訂單 ¶

商店端在確認訂單需要作廢時，可在未付款時將訂單取消。信用卡交易則可在會關帳前進行取消訂單。

POST /1177payment/api/cancel_order
RequestsCancelOrder
Headers
Content-Type: application/x-www-form-urlencoded
Body
merchantnumber=201387&ordernumber=20181031019&merchantordernumber=MO20181031019&amount=2018&currency=TWD&paymenttype=creditcard&paytitle=%E6%B8%AC%E8%A9%A6%E4%BA%A4%E6%98%93&duedate=20181231235959&id=A123456789&payphone=0911123456&cardnumber=4023100079998012&cardexpiry=201810&cardcvc2=705&transmode=N&consumerip=192.168.11.56&checksum=C51BACBB2B03355A0DB44018CF54CDAE7937333B80C32A713C0BF03379086CF2&timestamp=20181031193049

{
  "rc": "0",
  "ordernumber": "20181031022",
  "rm": "OK",
  "merchantnumber": "201387"
}

RefundReversal 退款取消 ¶

商店端在不慎按錯退款，或是要修改退款金額，或是消費者改變想法需要取消退款，可在該筆退款尚未關帳前進行退款取消。 已關帳的退款無法進行退款取消，僅可重新交易。

目前僅信用卡支援退款取消功能。
