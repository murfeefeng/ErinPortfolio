    PayPage API for Credit Card

    API Endpoint

    PayPage 是交易付款頁面，讓商店不用處理各個支付工具的選擇或填寫，也不用處理信用卡的相關規範(例如須通過PCI)，只要以固定的串接資料即可使用各支付工具。
    Paypage也允許從商店端指定支付工具，此時除固定參數外，需另外帶該支付工具的特殊參數，即可簡化支付流程。
    API規格中的編碼皆為utf-8。

    Payment API ¶

    Payment 交易傳入 ¶

    商店端在結帳時將訂單交易資料以頁面form post型式送到此API，經驗證正確後就會顯示可使用的支付工具選項。

    POST /1177payment/paypage/payment
    Requestspayment
    Headers
    Content-Type: application/x-www-form-urlencoded
    Body
    merchantnumber=117787&ordernumber=20181008140828&merchantordernumber=20181008140828&amount=2018&currency=TWD&paymenttype=atm&paytitle=PayPage測試交易&paymemo=PayPage測試交易&duedate=20181107140828&id=&payphone=**********&timestamp=20181008140828&checksum=789A811B2AEA001ED8F7DCD1FAFA3DCAC2CA70ACB77DEBD7B189DB2B62AA9DC7
    Responses200
    Body
    消費者瀏覽器頁面從商店端轉導到1177端，所以不會有回覆參數。

    Payment 交易傳入
    POST/1177payment/paypage/payment

    Request 參數

    名稱 型別 必要參數® 說明
    merchantnumber String(10) R 商店編號，由1177Tech配發，只可英數字
    ordernumber String(30) R 訂單編號，必須是唯一值，若有重複則會回覆訂單編號重複的錯誤，只可英數字和特殊符號[_-]。
    merchantordernumber String(30) O 商店訂單編號，此欄位不做唯一值的確認，供商店做訂單的對照使用，若不帶則系統分配的預設值為訂單編號。
    amount Number(11,2) R 訂單金額，可接受小數到小數點後2位，小數部分主要是為了多幣別時使用，當幣別為新台幣時接受金額只到整數，若有指定小數則無條件捨去。
    currency String(3) O 幣別，預設為TWD。(目前僅接受台幣)
    paymenttype String(20) O 此筆訂單所要使用的支付工具，若不帶此參數則表示到PayPage後再選擇支付工具。目前支援的參數值有creditcard。
    paytitle String(50) O 付款說明，此說明主要會顯示在某些支付工具的收款單位介面上，例如在超商的Kiosk機台上或是銀行WebATM的介面上。
    paymemo String(200) O 付款註記，在某些應用時需要系統多註記資訊以便日後查詢使用。
    duedate Number(14) O 訂單有效期限，若未帶此參數則會以各商店預設的期限為準(於商家資料設定)，有效期限會依選定的支付工具套用，例如信用卡交易可以到秒(yyyyMMddHHmmss)，超過這個時間才送出的訂單會被視為無效；而虛擬帳號因為銀行的期限到日(yyyyMMdd)，所以會自動判定為指定時間的當天，例如指定20181008153400，則此訂單在20181009即失效。
    id String(10) O 身分證字號或統編。
    lang String(2) O 頁面使用語言(多國語言)，目前支援的參數值有ch, jp, en
    payphone Number(12) O 消費者聯絡電話，可以是手機或市話，市話需加區碼。
    timestamp Number(14) R 時間戳記，格式為yyyyMMddHHmmss，若目前時間與時間戳記相差超過3分鐘即失效。
    checksum String(64) R 驗證碼，用以驗證傳送資料是否遭竄改。

    信用卡測試卡號

    卡別 卡號 到期日MM/YY 卡片後三碼 支援3D
    JCB 3560721234560032 12/30 000 Y
    VISA 4682358976116102 03/30 057 Y
    Master 5543850005881504 11/30 934 Y
    JCB 3565661234560027 12/30 000 N

    Master 5521970079998012 10/30 039 N

    驗證碼計算方式：

    將參數名稱依字母排序後，以key=value&key=value&…的格式組合好後(若沒有值則不加入驗證)，最後直接補上交易密碼(transcode，申請商店後由系統核發)，以utf-8取byte值後進行sha256。

    範例： 交易密碼(transcode)為 3710F366CD0329255166A5B30216E176

    資料為

    amount=200.5

    merchantnumber=201387

    ordernumber=54613351

    paytitle=這是測試

    paymemo=

    資料排序組合後為 “amount=200.5&merchantnumber=201387&ordernumber=54613351&paytitle=這是測試”

    準備進行hash的明文為 “amount=200.5&merchantnumber=201387&ordernumber=54613351&paytitle=這是測試3710F366CD0329255166A5B30216E176”

    產出的驗證碼為 “6B67D324F6A4396F0857192AE05946C443D28C7BC8E9C5CDB305F26671805A84”

    電文範例.
    “amount=200.5&merchantnumber=201387&ordernumber=54613351&checksum=6B67D324F6A4 396F0857192AE05946C443D28C7BC8E9C5CDB305F26671805A84&paytitle=這是測試”

    Response 參數

    消費者瀏覽器頁面從商店端轉導到1177端，所以不會有回覆參數。

    Payment 頁面導回 ¶

    在交易完成時(無論成功或失敗)，在結果頁面上會有"回首頁"的按鈕，按下後會將處理結果(不一定完成繳費)以頁面form post方式導回商店接收頁面，商店可用程式接收導回的參數以做更新和顯示。

    頁面導回商店端不代表交易已成功或失敗，僅代表線上交易告一段落，需待收到交易異步通知才能確認交易結果是否成功。

    POST /商店端接收網頁網址
    Requestspayment
    Headers
    Content-Type: application/x-www-form-urlencoded
    Body
    merchantnumber=117787&ordernumber=20181008140828&merchantordernumber=20181008140828&amount=2018&currency=TWD&paymenttype=atm&paytitle=PayPage測試交易&paymemo=PayPage測試交易&duedate=20181107140828&id=&payphone=**********&timestamp=20181008140828&checksum=789A811B2AEA001ED8F7DCD1FAFA3DCAC2CA70ACB77DEBD7B189DB2B62AA9DC7
    Responses200
    Body
    消費者瀏覽器頁面從商店端轉導到1177端，所以不會有回覆參數。

    ordernumber String(30) R 訂單編號，必須是唯一值，若有重複則會回覆訂單編號重複的錯誤，只可英數字和特殊符號[_-]。
    paymenttype String(20) O 此筆訂單所選擇使用的支付工具，若在尚未選擇時就已錯誤則不會回覆此欄位。
    timestamp Number(14) R 時間戳記，格式為yyyyMMddHHmmss，若目前時間與時間戳記相差超過3分鐘即失效。
    checksum String(64) R 驗證碼，用以驗證傳送資料是否遭竄改。

    Response 參數

    消費者瀏覽器頁面從1177端轉導到商店端，所以不會有回覆參數，僅需顯示網頁。

    Payment 交易異步通知 ¶

    在完成繳費時會將交易結果以form post方式通知商店接收程式，商店可用程式接收導回的參數以做更新。

    此為API通知，不需要回覆網頁，但需要回覆特定回覆格式資料讓1177Pay能確認商店端已收到無誤，若未收到商店端的接收成功回覆碼，則系統會在每隔30分鐘重試通知，共重試10次(5小時)後停止通知，商店若仍未接收成功而無法確認該筆訂單結果可至管理介面查詢。

    POST /商店端接收程式網址
    Requestspayment
    Headers
    Content-Type: application/x-www-form-urlencoded
    Body
    rc=0&rm=回覆訊息&merchantnumber=201387&ordernumber=20181017091801&paymentnumber=20181017091802&paymenttype=atm&paidamount=250&timestamp=20181017091900&checksum=E61FDBFA5D346E2E18DBDC163ACE2CA3DC26D095CBFF3DD92260CE9F76CAB39F
    Responses200
    Headers
    Content-Type: application/json
    Body
    {
  "rc": "0"
}

    Payment 交易回覆-交易異步通知
    POST/商店端接收程式網址

    Request 參數

    名稱 型別 必要參數® 說明
    rc String(4) R 回覆碼
    rm String(50) O 回覆訊息
    merchantnumber String(10) R 商店編號，由1177Pay配發，只可英數字
    ordernumber String(30) R 訂單編號，必須是唯一值，若有重複則會回覆訂單編號重複的錯誤，只可英數字和特殊符號[_-]。
    paymentnumber Number(14) R 付款序號，以此序號確認是否為重複繳款：若收到多次通知但序號相同則為重複銷帳，不須理會；若收到多次通知且序號不同則為重複繳款，必須記錄，此與帳務相關。
    paymenttype String(20) O 此筆訂單所選擇使用的支付工具，若在尚未選擇時就已錯誤則不會回覆此欄位。
    paidamount Number(14) O 實繳金額，交易成功才會回覆
    approvecode String(6) O 授權碼，選擇信用卡付款(paymenttype=creditcard)且交易成功時才會回覆
    ins_down_amt Number(9) O 分期首期金額，選擇信用卡分期付款(paymenttype=creditcard)且交易成功時才會回覆
    ins_each_amt Number(9) O 分期每期金額，選擇信用卡分期付款(paymenttype=creditcard)且交易成功時才會回覆
    red_offset_amt Number(9) O 紅利扣抵金額，選擇信用卡紅利付款(paymenttype=creditcard)且交易成功時才會回覆
    red_used_point Number(9) O 紅利扣抵點數，選擇信用卡紅利付款(paymenttype=creditcard)且交易成功時才會回覆
    timestamp Number(14) R 時間戳記，格式為yyyyMMddHHmmss，若目前時間與時間戳記相差超過3分鐘即失效。
    cardlinkid String(64) O 信用卡token，僅信用卡交易成功才會回覆
    checksum String(64) R 驗證碼，用以驗證傳送資料是否遭竄改。(參照：驗證碼計算方式)

    Response 參數

    名稱 型別 必要參數® 說明
    rc String(4) R 回覆碼

    Control API ¶

    CancelOrder 取消訂單 ¶

    商店端在確認訂單需要作廢時，可在未付款時將訂單取消。信用卡交易則可在會關帳前進行取消訂單。

    POST /1177payment/api/cancel_order
    RequestsCancelOrder
    Headers
    Content-Type: application/x-www-form-urlencoded
    Body
    merchantnumber=117840&ordernumber=1575944586&consumerip=192.168.11.77&checksum=DFAADA8CCFAC9B334EBFB6DA79526B7370737A662ADFF2A717E290DFE2857C1C&timestamp=20191211105148
    Responses200
    Body
    {
  "rc": "0",
  "paymenttype": "creditcard",
  "merchantnumber": "117840"
}

    CancelOrder 取消訂單
    POST/1177payment/api/cancel_order

    Request 參數

    名稱 型別 必要參數® 說明
    merchantnumber String(10) R 商店編號，由1177Tech配發，只可英數字
    ordernumber String(30) R 訂單編號，必須是唯一值，若有重複則會回覆訂單編號重複的錯誤，只可英數字和特殊符號[_-]。
    timestamp Number(14) R 時間戳記，格式為yyyyMMddHHmmss，若目前時間與時間戳記相差超過3分鐘即失效。註:時間以台北時間(GMT+8)為準
    consumerip String(15) O 消費者IP

    Refund 退款 ¶

    商店端在確認訂單需要做退款時，可在信用卡交易未關帳時進行退款。

    目前僅信用卡支援退款功能。

    POST /1177payment/api/refund
    RequestsRefund
    Headers
    Content-Type: application/x-www-form-urlencoded
    Body
    merchantnumber=117840&ordernumber=20191209094542&refundamount=10&consumerip=192.168.11.77&checksum=4B43896939FFD203C76FDB3D38A886DC0D261FE135BFA3AACA48C9A3868505D2&timestamp=20191211105743
    Responses200
    Body
    {
  "rc": "0",
  "refundtime": 20191211105744,
  "refundnumber": "190",
  "refundamount": "10",
  "paymenttype": "creditcard",
  "merchantnumber": "117840"
}

    RefundReversal 退款取消 ¶

    商店端在不慎按錯退款，或是要修改退款金額，或是消費者改變想法需要取消退款，可在該筆退款尚未關帳前進行退款取消。 已關帳的退款無法進行退款取消，僅可重新交易。

    目前僅信用卡支援退款取消功能。

    POST /1177payment/api/refund_reversal
    RequestsRefundReversal
    Headers
    Content-Type: application/x-www-form-urlencoded
    Body
    merchantnumber=117840&ordernumber=20191209094542&refundnumber=190&checksum=17471D9B02FF78FA23B3953135DFD643F1FB2366ED672E72B3E0897A66D49F62&timestamp=20191211135729
    Responses200
    Body
    {
  "rc": "0",
  "refundreversaltime": 20191211135731,
  "paymenttype": "creditcard",
  "merchantnumber": "117840"
}
